# Solar Panel Anomaly Detection - Raspberry Pi Daily Check Guide

This directory contains the script designed to run as a **daily automated check** on a Raspberry Pi local server. 

The script ingests a daily log (already parsed structured JSON data) for an ESP32 device, tracks its historical telemetry in a local SQLite database to construct the 30-day chronological rolling sequence, and runs inference via the PyTorch LSTM neural network to detect hardware faults early.

---

## 📂 Deployment Package Structure

To run the daily check on your Raspberry Pi, you need to copy the following files:

```text
RaspberryPi/
│
├── daily_check.py          # Daily check CLI program
├── requirements.txt        # Python library dependencies
├── device_history.db       # Local SQLite Database (Autogenerated on first run)
│
└── models/                 # Neural Network Weights & Configurations (Copy from PipeLine/models/)
    ├── lstm_fault_detector.pth
    ├── model_config.pkl
    ├── scaler.pkl
    ├── label_encoder.pkl
    └── feature_columns.pkl
```

> [!IMPORTANT]
> **Before running the script**, make sure you copy the entire `models/` directory from the `PipeLine/models/` folder into your `RaspberryPi/` folder.

## ⚙️ Setup and Installation on Raspberry Pi

### 1. Python Environment Setup
SSH into your Raspberry Pi, navigate to the folder where you copied the project, and create a virtual environment:

```bash
cd RaspberryPi
python3 -m venv .venv
source .venv/bin/activate
```

### 2. Dependency Installation
Install dependencies inside the virtual environment:

```bash
pip3 install -r requirements.txt
```

> [!TIP]
> On Raspberry Pi OS (especially older ARM models), compiling `torch` or `scikit-learn` from source using `pip` can be slow or run out of memory. 
> - If you run into issues installing `torch`, install it via your system package manager or download precompiled wheels matching your OS architecture (e.g. 32-bit or 64-bit ARM).
> - On Raspberry Pi OS, you can also install system libraries with `sudo apt install python3-torch python3-pandas python3-numpy python3-sklearn` if you prefer to use the global python environment rather than a virtual environment.

## 🚀 Usage Guide

### 1. Running a Daily Check
To check the status of a specific ESP32 datalogger, run `daily_check.py` passing the **parsed daily log** and the **device serial number**:

```bash
python3 daily_check.py <path_to_daily_log.json> <device_serial_number>
```

#### Example Command:
```bash
python3 daily_check.py parsed_log_today.json ESP32_SN_87920
```

### 3. Clear/Reset Device History
If you replace the battery/controller on an ESP32 or are testing, you can reset its historical records in the database by passing the `--clear-history` flag:

```bash
python3 daily_check.py parsed_log_today.json ESP32_SN_87920 --clear-history
```

### 4. Direct Python Integration (Function Call)
For team workflows where you want to call this system programmatically as a Python function, you can import and run `run_daily_check` directly in your scripts:

```python
from daily_check import run_daily_check

# 1. Define your daily telemetry data directly (dict or list)
daily_telemetry = {
  "vbat_min_v": 12.7,
  "vbat_max_v": 13.8,
  "ah_charge_ah": 19.1,
  "ah_load_ah": 14.8,
  "vpv_max_v": 18.0,
  "ipv_max_a": 3.0,
  "soc_pct": 90.0,
  "text_max_c": 31.0,
  "text_min_c": 22.0,
  "night_h": 10.0
}

# 2. Execute the function call passing the telemetry and serial number
result = run_daily_check(
    daily_data=daily_telemetry,
    serial_number="ESP32_SN_87920"
)

# 3. Read predictions and stats directly
print("Predicted Status:", result["prediction"]["anomaly_label"])
print("Confidence Score:", result["prediction"]["confidence"])
print("Action Mapped:", result["prediction"]["corrective_action"])
```

#### Arguments of `run_daily_check`:
- `daily_data`: (`dict`, `list`, `str`) The daily log data. Can be a python dictionary representing a single day, a list of daily dictionaries, a JSON string, or a JSON file path.
- `serial_number`: (`str`) Unique serial number identifying the ESP32 logger.
- `db_path`: (`str`, optional) Path to custom SQLite database file. Defaults to `device_history.db` in the script's directory.
- `model_dir`: (`str`, optional) Path to custom model directory. If None, resolves standard locations automatically.
- `clear_history`: (`bool`, optional) Reset/clear stored history for this serial number prior to evaluation.

#### Return Format:
Returns a Python dictionary matching the `--json` output format.

## 🕒 Automating Daily Checks with Cron

To schedule this script to run automatically when a device posts its daily telemetry log:

1. Open your Raspberry Pi crontab editor:
   ```bash
   crontab -e
   ```
2. Add a rule to execute the script daily (for example, at 23:30 (11:30 PM) every day):
   ```text
   30 23 * * * /home/pi/RaspberryPi/.venv/bin/python /home/pi/RaspberryPi/daily_check.py /home/pi/logs/today_ESP32_SN_1.json ESP32_SN_1 >> /home/pi/logs/check_SN_1.log 2>&1
   ```
